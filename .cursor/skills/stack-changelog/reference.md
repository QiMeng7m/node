## package.json → §2.1 层级对照

| package 名 / 模式 | 建议「层级」列 |
|-------------------|----------------|
| `react` | 前端框架 |
| `typescript`（client） | 前端语言 |
| `vite` | 构建工具 |
| `antd` | UI 组件库 |
| `react-router-dom` | 路由 |
| `express` / `fastify` / `koa` | 后端框架 |
| `typescript`（server） | 后端语言 |
| `@prisma/client` / `prisma` | ORM |
| `better-sqlite3` / `pg` | 数据库 |
| `pm2` / `nginx`（文档层） | 部署 |

版本列写 `package.json` 中的 semver 范围（如 `^5.1`），不要写 lock 里解析后的精确补丁号，除非用户要求。

## §9 摘要写法示例

**差（缺理由）：**
> 接入 Prisma

**好（自包含）：**
> 接入 SQLite + Prisma 替代 posts 硬编码；理由：Phase 1 需要持久化与迁移，SQLite 零运维适合 MVP

**替换：**
> Express 保留，新增 Zod 校验层；理由：统一请求体验证，减少手写 if 判断

**否决：**
> 否决 LangChain 接入（暂缓）；理由：MVP 仅需直连 API 流式，复杂度超需求

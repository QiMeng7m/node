## 1. 产品概要

| 项 | 内容 |
|----|------|
| 产品名 | AI 工作台（暂定） |
| 目标用户 | 10～50 人小团队 |
| 核心场景 | 自由对话、技术问答、文档生成、图片分析 |
| 管理员能力 | 添加 Provider/密钥、注册模型、管理 Feature 与用户配额 |
| 技术边界 | 模型 API 外置；Key 仅服务端；OpenAI 兼容协议 |

---

## 2. 模块地图

```
M1 认证与权限
M2 Provider / 模型管理（管理员）
M3 Feature 场景引擎
M4 对话与流式输出
M5 会话管理
M6 图片上传与 Vision
M7 用量与限流
M8 管理后台
M9 个人站集成（可选）
```

## 3. 需求分解表

### M1 认证与权限

| ID | 功能点 | 用户故事 | 优先级 | 版本 | 验收标准 |
|----|--------|----------|--------|------|----------|
| M1-01 | 用户登录 | 作为成员，我用邮箱密码登录以使用 AI | P0 | v0.4 | 正确凭证返回 token；错误提示不泄露细节 |
| M1-02 | 用户注册 | 作为新成员，我注册账号（若开放） | P1 | v0.4 | 邮箱唯一；密码强度校验 |
| M1-03 | 角色区分 | 作为管理员，我拥有 admin 权限 | P0 | v0.4 | admin 可访问 `/admin` 与 `/api/admin/*` |
| M1-04 | 会话保持 | 作为用户，刷新页面后仍保持登录 | P0 | v0.4 | JWT 未过期则 `/api/auth/me` 返回用户信息 |
| M1-05 | 账号禁用 | 作为管理员，我禁用某用户 | P1 | v1.0 | 禁用后登录失败；已有 token 下次请求 403 |
| M1-06 | 开发期免鉴权 | 作为开发者，本地调试时可跳过登录 | P2 | dev | 环境变量 `AUTH_DISABLED=true` 时注入 mock 用户 |

**依赖**：无  
**阻塞**：M4、M5、M7、M8（上线前）

### M3 Feature 场景引擎

| ID | 功能点 | 用户故事 | 优先级 | 版本 | 验收标准 |
|----|--------|----------|--------|------|----------|

| M3-05 | 内置 Feature 包 | 作为用户，开箱即用 5+ 场景 | P0 | v1.0 | 见 §4 内置清单 |

**内置 Feature 清单（v1.0）**

| slug | 名称 | 类别 | modelPolicy | 输入形态 |
|------|------|------|-------------|----------|
| `free-chat` | 自由对话 | chat | free | 纯文本 |
| `tech-qa` | 技术问答 | code | recommended | 纯文本（可贴代码） |
| `doc-generate` | 文档生成 | doc | recommended | 表单：标题、类型、要点 |
| `text-polish` | 文本润色 | doc | recommended | 文本 + 风格 select |
| `summarize` | 内容摘要 | doc | recommended | 长文本 |
| `image-analyze` | 图片分析 | image | locked | 图片 + 可选问题 |

| ID | 功能点 | 用户故事 | 优先级 | 版本 | 验收标准 |

## 6. 跨模块业务规则

| 规则 ID | 描述 |
|---------|------|
| BR-01 | API Key 永不出现在前端、日志、API 响应 |
| BR-02 | `POST /api/chat` 的 messages 不含 client 构造的 system；system 由服务端根据 featureId 注入 |
| BR-03 | 同一 session 内 model 变更只影响后续 assistant 生成 |
| BR-04 | Feature `locked` 时忽略用户 model 参数，强制 defaultModelId |
| BR-05 | 图片消息仅允许 `supportsVision=true` 的模型 |
| BR-06 | 软删除 session 不物理删除 message（便于审计，可选硬删策略） |
| BR-07 | 开发环境 v0.4 前允许无 auth；生产必须 auth |
